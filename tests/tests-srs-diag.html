SRS / PSA Diagnostic Tests

Investigating T09 · T13 · T14 · T15 — Starter Rate for Savings & PSA interaction with PA

Purpose: These tests systematically probe the SRS and PSA logic in calcIncomeTaxDetailed to confirm or refute the hand-calculated expected values for the four disputed scenarios from the previous session. Tests are written so that a failure points to the exact sub-expression that differs from the spec. All expected values are from independent Node.js calculation, not from the original (possibly wrong) spec sheet.

SRS rule: srsAvail = max(0, srsLimit(£5,000) − nsNet). SRS covers savings income after PA, up to the available headroom. PSA tier is then determined by taxableForPSA = nsNet + savNet + divNet vs band widths (£37,700 / £112,570).

Tolerance: £0.02 for all pure tax function tests.
