SRS & PSA Tests

Savings Rate Band (£5k @ 0%) phaseout · PSA tier boundaries · Dividend / PSA interaction

HMRC rules (2026-27): SRS allows up to £5,000 savings at 0%, reduced £1-for-£1 by non-savings income above PA. SRS=0 when non-savings >= £17,570. PSA: £1,000 basic rate, £500 higher rate, £0 additional. Income ordering: non-savings then savings then dividends. Tolerance: £0.02 pure function.
KNOWN BUG - PSA tier determination (calculator.js): The engine computes taxableForPSA = nsNet + savNet + divNet and compares against the basic band (£37,700). This incorrectly includes dividend income. HMRC determines PSA tier by whether savings income sits within the basic band - dividends stacked above should not influence this. Result: a basic-rate taxpayer with significant dividends gets the £500 higher-rate PSA instead of £1,000, causing a £100/year overcharge per affected person. Tests marked with a warning symbol document the engine's current (wrong) output.
